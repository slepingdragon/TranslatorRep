# DR Reconciliation — TranslatorRep PRD

## Overall verdict

The PRD captures the DR's linguistic findings with high fidelity. §5 Translation Quality Requirements is a custom section that explicitly elevates the Gemini system prompt to a versioned, first-class component, enumerates all DR preservation targets, and codifies the paired-review quality acceptance test. The PRD also correctly preserves Bania's resolved choice to keep the harder Sundanese framing in §10.2 (with an explicit NOTE FOR PM blocking drift in downstream planning). The main residual issue is one missed DR recommendation — the translation-quality review tool — which appears nowhere in the PRD as a feature or open question.

## Gaps (DR content the PRD dropped or under-captured)

- **Translation quality review tool not surfaced.** DR §Recommendations explicitly calls for "a translation quality review tool as part of v1 — even a simple Compose screen that lets Bania review the last 20 utterances and mark which translations were good/bad/off-tone gives him a real feedback channel." This is a concrete v1 capability recommendation from DR. The PRD has no FR for it, no open question about it, and §5.3's quality acceptance test contemplates a one-shot pre-launch review rather than an ongoing in-app feedback loop. FR-21 (transcript history) is adjacent but is a privacy feature, not a quality-rating capture mechanism. Either add a v1 FR for caption rating, or downgrade to an explicit open question / v2 candidate so it isn't silently lost.
- **DR's iteration cadence partially captured.** §5.1 mentions "review after the first 10 real conversations; revise based on actual error patterns," which echoes DR §Implementation Notes. But DR's emphasis that the prompt is "the highest-leverage v1 artifact you can iterate on" — and that iterating it is cheap because it's plain text — isn't fully reflected as a rhythm in the PRD. Adequate, but light.
- **Prosody / sentence-final particle buffering (DR §7).** DR notes that emotional load lands at the END of Indonesian sentences and that chunk-by-chunk streaming "systematically misses the particle"; recommends buffering to sentence-final before emitting. The PRD's FR-12 VAD section and FR-18 Partial Caption section don't reflect this constraint. It's arguably implementation-level, but it shapes whether Partial Captions can ever be reliably translated, which is a product-visible question.
- **DR's "soft recommendation to update the brief" not closed out.** DR §Update to the CB Brief asks Bania to confirm softening the brief's Sundanese gap framing. PRD §10.2 resolves the question (rejected DR's softening) but doesn't note whether the brief itself was actually updated or left in original form — minor traceability gap.

## Drift (where the PRD wandered from Bania's resolved positions)

- **No Sundanese framing drift detected.** §10.2 explicitly preserves the harder "real and visible gap" / "not optional, just sequenced after v1 ships" framing with an explicit NOTE FOR PM ("Bania explicitly chose to keep this harder framing for honesty even though DR's lexical-insertions finding softens it somewhat. Preserve the 'not optional' sequencing language in v2 planning."). §2.3 Non-Users uses neutral language ("Sundanese insertions handled via the v1 translation prompt; full Sundanese clauses are flagged not fully supported until v2") that's consistent with the harder framing. §5.2 references the Sundanese token set without re-introducing DR's optimistic "v1 handles ~90% of her code-switching" claim. Consistent throughout.
- **No register / quality framing drift detected.** §5.2 enumerates preservation targets exactly as DR §5.2 specified (14 particles, register signals, gender neutrality default to "they," honorific stripping, religious expression preservation, indirect refusals, Gen-Z slang).

## Over-commitment (DR specifics that don't belong in PRD)

- **Full system prompt text correctly NOT inlined.** PRD references it by location (`prompts/id-en-v1.md` in Cloud Run repo) rather than reproducing the ~2,200-token prompt. Correct restraint — the prompt is an implementation artifact.
- **§5.2 enumerates the 14 particles by name.** This is borderline. Arguments for keeping in PRD: the particle list defines the testable preservation contract and is shorter than the full prompt; downstream Epics/Stories need it as the acceptance surface. Arguments against: the canonical list lives in DR §1 and could be referenced rather than duplicated, reducing the risk of the two documents drifting. Current treatment is acceptable but worth flagging — if the DR particle list is ever revised, the PRD list must be updated in lockstep. Same consideration for the "≥12 high-frequency Sundanese tokens" and "~20 high-frequency Gen-Z slang items" references — PRD wisely uses counts + cross-references rather than re-listing.
- **No other over-commitment.** PRD does not inline the EN↔ID failure-mode table, the source citations, the slang map, the endearment list, or the prosody/filler rules — all correctly left in DR as the linguistic reference companion.
