# `/firebase/` — Firebase CLI Workspace

> **Purpose:** This directory is the Firebase CLI workspace for the TranslatorRep project. Holds Firestore security rules, App Check provider notes, and CLI config. Owned by Story 1.4 (Android scope) + Story 1.4b (iOS scope, future).

---

## What lives here

| File | Purpose |
|---|---|
| `firebase.json` | CLI config — declares Firestore (no Functions, no Hosting, no Storage in v1 scope) |
| `firestore.rules` | Security rules deployed to Cloud Firestore. Source of truth. |
| `firestore.indexes.json` | Composite-index config. Empty in v1; populated as queries that need indexes get added. |
| `.firebaserc` | (Generated by `firebase use --add` — not in this PR; created by Bania during Story 1.4 Phase 0.9.) Aliases the local dir to the Firebase project. |
| `appcheck/android-providers.md` | Play Integrity provider setup notes (per Story 1.4 AC-3) |
| `appcheck/ios-providers.md` | DeviceCheck provider setup notes (stub — populated in Story 1.4b) |

---

## What does NOT live here

- ❌ **`google-services.json`** (Android Firebase credentials) — lives at `android/app/google-services.json`, gitignored. Downloaded from Firebase console; per-developer.
- ❌ **`GoogleService-Info.plist`** (iOS Firebase credentials) — will live under `ios/TranslatorRep/Resources/GoogleService-Info.plist` when Story 1.4b lands; gitignored.
- ❌ **Cloud Functions** — explicitly out of v1 scope per architecture decision. The auth-proxy (Story 1.3) on Oracle VM replaces the need for Functions.

## Common commands

Run all from this directory (`cd firebase`):

| Command | What it does |
|---|---|
| `firebase login` | One-time CLI auth via browser |
| `firebase use --add` | Link this dir to a Firebase project (run once on a fresh clone) |
| `firebase use` | Show currently-linked project |
| `firebase deploy --only firestore:rules` | Push `firestore.rules` to Cloud Firestore. Use after editing rules. |
| `firebase emulators:start --only firestore` | Local emulator for testing rules without touching prod |
| `firebase deploy` | DON'T USE — would try to deploy Functions/Hosting/etc. that aren't configured. Always scope with `--only`. |

---

## Rules-change workflow

1. Edit `firestore.rules`.
2. Test locally against the emulator (optional but recommended): `firebase emulators:start --only firestore` + run rules unit tests.
3. Test in the Firebase console's Rules Playground (Firebase console → Firestore → **Rules** → **Rules playground**) for ad-hoc scenarios.
4. Deploy: `firebase deploy --only firestore:rules`.
5. Verify deployment in the Firebase console → Firestore → **Rules** tab (the timestamp updates).
6. Commit the rules file. CI doesn't auto-deploy — deployment is manual + reviewed.
